## 2. Правила роботи зі сховищем (Zone of Responsibility)
* `01_Product/` — Зона бізнесу та дизайну. Тут лежать вимоги, флоу юзера, описи цільової аудиторії. 
* `02_Architecture/` — Технічна зона. C4 Model, секвенс-діаграми. 
* `03_ADR/` — Лог архітектурних рішень. 
* `04_API/` — Контракти взаємодії між Python, Electron та бекендом.

## 3. Як ми використовуємо формати
* **`.md` (Markdown):** Основний формат для всього тексту, інструкцій та коду. Діаграми пишемо через Mermaid. 
*  **`.canvas` (Obsidian Canvas):** ТІЛЬКИ для високорівневих схем (наприклад, загальна архітектура системи). Заборонено дублювати сюди текст з `.md` файлів, щоб уникнути конфліктів при мерджі.
---
## 4. Що таке ADR (Architecture Decision Record)?
Якщо ми приймаємо глобальне технічне рішення (наприклад, обрали WebRTC замість WebSocket для відео), ми фіксуємо це тут. Формат ADR:
1. Контекст (що вирішуємо). 
2. Рішення (що обрали). 
3. Наслідки (плюси/мінуси).
