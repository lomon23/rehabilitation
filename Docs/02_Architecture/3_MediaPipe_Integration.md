## Архітектура локального ML
Щоб уникнути перевантаження Main Thread браузера/Електрона і лагів UI, всі тензорні обчислення винесені в ізольований фоновий потік. Ми не використовуємо Python на клієнті.

1. **Технологія:** MediaPipe Pose (TypeScript API через WebAssembly).
2. **Ізоляція (Web Worker):** - `Main Thread` (React) відповідає тільки за захоплення кадрів з `getUserMedia()` та малювання інтерфейсу.
   - Кожен кадр конвертується в `ImageBitmap` і передається у `Web Worker`.
   - `Web Worker` проганяє кадр через MediaPipe, знаходить 33 координати (X, Y, Z) і повертає готовий масив назад у Main Thread.

## Вирішення апаратних проблем

* **Проблема Jitter (Смикання точок):** Модель може давати мікро-похибки на нерухомій людині.
  - *Рішення:* Використання EMA (Експоненційне ковзне середнє) у Web Worker перед розрахунком кутів для згладжування координат.
* **Калібрування простору (2D/3D):**
  - *Рішення:* Блокатор запису. Трекінг не ініціюється, поки модуль не підтвердить фронтальну/профільну позицію (похибка Z-координати лівого і правого плеча < X%).

## Розділення потоків (UI Overlay)
Відео та скелет — це різні сутності. Вони ніколи не мерджаться в один відеофайл.
- **Під час запису/дзвінка:** React малює прозорий `<canvas>` поверх `<video>`. Лінії малюються по координатах з Web Worker.
- **Toggle Mode:** Кнопка "Сховати скелет" просто застосовує `display: none` до тегу `<canvas>`. Відеопотік залишається недоторканим.
