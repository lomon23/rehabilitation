# Math & Coordinate Filtering

**Статус:** Draft

## 1. Фільтрація шумів (Anti-Jitter)
Сирі дані з MediaPipe містять мікро-флуктуації. Для стабілізації скелета використовується фільтр EMA (Exponential Moving Average).

- Кожна нова координата не застосовується миттєво. Вона "змішується" з попереднім станом.
- **Коефіцієнт Alpha ($\alpha$):** Налаштовується динамічно (наприклад, $0.5$). Нижче значення = більша плавність, але вища затримка.

**Математична модель EMA:**
$$EMA_t = \alpha \cdot X_t + (1 - \alpha) \cdot EMA_{t-1}$$

*Де $X_t$ — поточна сира координата з кадру.*

## 2. Розрахунок кутів (Vector Math)
Бізнес-логіка правильного виконання вправи базується на вимірюванні кутів між суглобами.

- Визначаються три точки: A (початок), B (вершина кута/суглоб), C (кінець).
- Формуються два вектори: $\mathbf{u}$ (від B до A) та $\mathbf{v}$ (від B до C).
- Кут $\theta$ обчислюється через арккосинус скалярного добутку векторів, поділеного на добуток їхніх довжин:

$$\theta = \arccos\left(\frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{u}\| \|\mathbf{v}\|}\right)$$

- Результат $\theta$ отримуємо в радіанах. Для подальшого виведення в UI він конвертується в градуси: $\theta \cdot \frac{180}{\pi}$.

## 3. Валідація видимості (Confidence Score)
Алгоритм відкидає обчислення, якщо показник `visibility` для будь-якої з трьох точок падає нижче `0.65` ($65\%$). У такому разі кадр ігнорується, щоб уникнути хибних результатів (наприклад, якщо рука перекрита тілом або вийшла за межі екрана).
