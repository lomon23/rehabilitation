# Схема даних: telemetry.json

**Статус:** Draft
**Тип файлу:** Специфікація внутрішнього зашифрованого файлу звіту (Solo Mode)
**Пов'язані файли:** [[2_Exercises_API]], [[3_Exercise_Logic]]

## Концепція
Файл містить повний масив біометричних точок пацієнта та розрахованих кутів, синхронізованих із часовою шкалою записаного відео файлу (`video_record.webm`). Фронтенд лікаря використовує цей JSON для рендеру графіки поверх відео без повторного запуску важких ML-моделей.

## Специфікація оптимізації об'єму ( Payload Optimization)

Щоб файл не важив десятками мегабайтів і не забивав Google Drive пацієнта:

1. **Фільтрація точок:** З 33 стандартних точок MediaPipe ми зберігаємо в JSON **тільки ті, що потрібні для конкретної вправи** (для присідань — індекси 11, 12, 23, 24, 25, 26, 27, 28). Обличчя, кисті та пальці відкидаються у Web Worker ще до формування пакету.
    
2. **Округлення:** Координати `x, y, z` та значення кутів округлюються до двох знаків після коми (`.toFixed(2)`). Точності в межах міліметра достатньо, а вага файлу падає в 3 рази.
